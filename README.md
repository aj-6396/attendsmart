# Class Mark - Official College Attendance System

Class Mark is a premium, institutional attendance monitoring system designed for modern colleges. It leverages **Geo-fencing**, **Multi-Class Cohorts**, and **Device Hardware Locking** to ensure 100% attendance accuracy.

## 🌟 Security Architecture

### 🛡️ 1. Hardware Device Locking
Students are locked to their registration device. 
- During registration, a unique hardware signature is stored.
- Students **cannot** log in from a friend's phone to fake attendance.
- Cross-device login attempts are instantly blocked and flagged.

### 📍 2. Geo-Fencing & Proximity
- Uses the **Haversine Algorithm** to calculate precise distances between teachers and students.
- Students must be within a calibrated radius of the classroom to mark attendance.
- Prevents "proxy" attendance from outside the college campus.

### 🔑 3. Dynamic OTP Verification
- Time-bound, 4-6 digit numeric OTPs generated by teachers.
- OTPs are strictly isolated to specific classrooms and cohorts.

## 🏫 Feature Set

### For Teachers (Faculty View)
*   **Class Grouping**: Create distinct cohorts (e.g., "Physics 101 - Semester 4") and generate unique **Join Codes**.
*   **Real-time Dashboard**: Watch the attendance list populate live as students punch in their OTPs.
*   **Manual Overrides**: Faculty can manually mark a student as present in edge cases.
*   **Institutional Exports**: One-click CSV downloads for official college records.
*   **Student Support**: Admin tools to reset student PINs or manage classroom enrollments.

### For Students (Learner View)
*   **Class Joining**: Enter a 6-character Join Code provided by the teacher to enroll in a subject.
*   **Secure Puncing**: Punch in the session OTP while physically inside the classroom.
*   **Academic History**: View attendance percentage and missing sessions isolated by subject/teacher.

## 🛠️ Technical Implementation

*   **Frontend**: React 19, Framer Motion (premium animations), Tailwind CSS.
*   **Backend**: Node.js & Supabase Edge Functions.
*   **Database**: PostgreSQL with Row Level Security (RLS) and Realtime WAL subscriptions.
*   **Security**: Device fingerprinting via browser localStorage & randomUUID tracking.

## 🚀 Deployment & Setup

### 1. Database Setup
Execute the contents of `supabase_migrations.sql` in your Supabase SQL Editor. This will:
1. Create the `classes` and `enrollments` architecture.
2. Enable RLS security policies.
3. Add the `device_id` column for hardware locking.

### 2. Environment Variables
Ensure your `.env` contains:
```env
VITE_SUPABASE_URL=your_project_url
VITE_SUPABASE_ANON_KEY=your_key
SUPABASE_SERVICE_ROLE_KEY=your_service_key (required for server-side auth)
```

### 3. Running Locally
```bash
npm install
npm run dev
```
